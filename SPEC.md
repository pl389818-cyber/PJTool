# PJTool SPEC（macOS）

更新时间：2026-05-01  
适用范围：`PJTool` 工具集（`录屏 / PiP 摄像 / 视频处理`）

## 1. 产品定位
`PJTool` 是一个 macOS 工具集，包含三大模块：
1. `录屏`
2. `PiP 摄像`
3. `视频处理`

本轮目标是：
1. 在不改变录屏与视频处理既有语义的前提下，将 `PiP 摄像` 对齐为“macOS 原生摄像头悬浮子工具”。
2. 将 `录屏` 对齐为“QuickTime 式主屏整屏录制工具”。

## 2. 信息架构（固定）

### 2.1 左侧菜单
左侧菜单固定三项，顺序固定：
1. `录屏`
2. `PiP 摄像`
3. `视频处理`

不再提供独立 `麦克风` 菜单。

### 2.2 页面职责
1. `录屏`：主屏整屏录制、录制控制、录制麦克风选择与监听。
2. `PiP 摄像`：悬浮摄像头预览与 PiP 专属音视频控制台。
3. `视频处理`：录制后视频拼接、剪切、导出。

## 3. 控制台布局约束（新增）
1. 左侧菜单栏默认启动宽度为 `200 px`。
2. 默认宽度必须在冷启动时强制生效，不受系统分栏历史状态覆盖。
3. 左侧菜单栏允许拖拽调整宽度，最小宽度不低于 `200 px`。

## 4. PiP 摄像模块（本轮强约束）

### 4.1 目标定义
`PiP 摄像` 是工具集中的一个 **macOS 原生摄像头悬浮工具**，用于桌面工作流下提供低延迟、可控、稳定的前置画中画预览体验。

### 4.2 本轮必须提供
1. 悬浮窗始终前置预览（跨 Space / 全屏辅助场景可见）。
2. 视频/音频设备手动选择（支持 iPhone Continuity 场景）。
3. 麦克风预览静音与实时音量电平。
4. 悬浮窗比例切换与自动放缩（`自动 / 16:9 / 4:3`）。

### 4.3 本轮不包含
1. PiP 页面录制入口。
2. PiP 页面导出入口。
3. PiP 页面语音转写入口。

说明：录制与导出能力仍由 `录屏` / `视频处理` 模块承载。

## 5. PiP 交互规范

### 5.1 页面定位
1. `PiP 摄像` 页面是工具控制台，不是通用设置页。
2. 主路径必须围绕“弹出 / 收起悬浮预览”组织。
3. 用户心智是“操作一个桌面悬浮工具”，不是“配置一个录屏附属开关”。

### 5.2 默认行为
1. 默认不自动启动摄像头预览。
2. `enableCameraPiP` 默认值为 `false`。
3. `bootstrap()` 仅做设备刷新与状态同步，不自动 `startPreviewIfNeeded()`。

### 5.3 顶部主入口
`PiP 摄像` 页面右上提供主按钮：
1. 未显示预览：`弹出 PiP 摄像`
2. 已显示预览：`收起 PiP 摄像`

点击规则：
1. 点击 `弹出 PiP 摄像`：内部自动启用 PiP，再调用预览弹出动作。
2. 点击 `收起 PiP 摄像`：仅隐藏预览，不反向关闭内部启用状态。
3. 页面不应把内部启用状态作为主操作暴露给用户。

### 5.4 页面组成
1. 顶部操作区：主按钮 + 当前 PiP 状态。
2. 设备区：视频设备、PiP 麦克风、刷新与权限恢复。
3. 监听区：静音、音量、实时电平。
4. 窗口区：`自动 / 16:9 / 4:3` 与当前布局摘要。
5. 诊断区：授权、枚举、发现源、应用内诊断。

### 5.5 比例与自动放缩
1. 比例枚举：`PiPAspectRatio.auto / sixteenByNine / fourByThree`。
2. `16:9` 与 `4:3` 切换沿用“保持高度重算宽度”。
3. `auto` 以可见性优先：
   - 以窗口中心为锚点；
   - 始终收敛在屏幕可见区域；
   - 满足最小尺寸与最大宽占比约束；
   - Space/屏幕变化时自动修正。

### 5.6 设备空状态与诊断
1. 无权限、无设备、设备离线必须给出明确状态文案，不允许静默失败。
2. 提供“刷新设备”“请求权限”“打开系统隐私设置”“运行应用内诊断”。
3. 诊断输出统一：`PASS / BLOCKED` 摘要与授权/枚举信息。

## 6. 窗口与设备技术约束（PiP）

### 6.1 悬浮窗前置
1. `NSPanel` 置顶层级：`mainMenu`。
2. `collectionBehavior` 至少包含：
   - `canJoinAllSpaces`
   - `fullScreenAuxiliary`
   - `moveToActiveSpace`
3. Space 切换、App 激活后执行 `orderFrontRegardless` 兜底。

### 6.2 设备选择
1. 视频设备与 PiP 麦克风设备分开选择。
2. 列表标签支持：`Continuity` / `Offline` / `Built-in`。
3. 枚举策略：
   - 优先 `AVCaptureDevice.DiscoverySession`
   - 空结果时回退 `AVCaptureDevice.devices(for:)`
4. 断连回退：设备离线时自动回退可用设备，并提示用户。

### 6.3 监听语义
1. `PiPAudioPreviewConfig.isPreviewMuted` 与 `previewVolume` 仅影响预览监听。
2. 不改变录制导出音轨语义。
3. 实时电平来自 `cameraEngine.previewAudioLevel`。

## 7. 三模块现状语义（保持不变）

### 7.1 录屏模块
1. 仅支持 `macOS 主屏整屏录制`，不再提供区域录制。
2. 点击开始录屏后隐藏主窗口。
3. 录制中显示独立悬浮控制器，主体为相机图标，可拖动。
4. 悬浮控制器标题栏提供停止方块按钮，点击后结束录屏。
5. 录制控制支持主窗口、菜单栏、悬浮控制器共享状态。
6. 录制麦克风可手选并显示监听电平。
7. 正常停止、启动失败、异常停止均自动恢复主窗口。
8. 录制时优先排除 `PJTool` 自身应用窗口；若排除失败，需提示录屏控制窗可能进入画面。

### 7.2 视频处理模块
1. 导入视频采用“拼接语义”（非叠加语义）。
2. 拼接支持插入到录屏时间线任意时间点。
3. 剪切采用“删除段模型”的多段剪切。
4. 支持导出与验证报告。

## 8. 数据模型/接口（当前实现对齐）
1. `SettingsSection`：`recording / pipCamera / videoProcessing`。
2. `PiPAspectRatio`：`auto / sixteenByNine / fourByThree`。
3. `PiPLayoutState`：`normalizedRect + aspectRatio`。
4. `PiPAudioPreviewConfig`：`isPreviewMuted + previewVolume`。
5. `AppCoordinator`：
   - `statusMessage`（录屏主状态）
   - `pipStatusMessage`（PiP 独立状态）
6. `RecordingRequest`：仅承载主屏录制所需的麦克风、摄像头与 PiP 合成参数。

## 9. 非目标
1. 本轮不重构录屏核心状态机。
2. 本轮不改变视频处理业务语义。
3. 本轮不引入 PiP 页面录制/导出/转写入口。

## 10. 验证策略（当前执行）

2. 逻辑校验：`Scripts/run_logic_checks.sh`。
3. 实机校验：
   - iPhone Continuity 设备可见性与切换；
   - PiP 前置显示、拖拽缩放、比例切换；
   - 录屏主窗口隐藏/恢复、悬浮控制器拖拽与标题栏停止方块；
   - 无设备/无权限提示链路。
