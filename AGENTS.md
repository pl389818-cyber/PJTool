# PJTool AGENTS 约束（macOS）

更新时间：2026-05-01

## 1. 产品边界（固定）
`PJTool` 是工具集，固定三模块：
1. `录屏`
2. `PiP 摄像`
3. `视频处理`

## 2. 本轮核心目标
本轮双目标并行：
1. 在不改变录屏/视频处理语义的前提下，`PiP 摄像` 对齐为“macOS 原生摄像头悬浮子工具”。
2. `录屏` 对齐为“QuickTime 式主屏整屏录制工具”。

## 3. PiP 模块强约束

### 3.1 产品定义
1. `PiP 摄像` 是独立子工具，不是录屏附属面板。
2. 目标是在桌面工作流中提供低延迟、可控、稳定的前置画中画体验。
3. 页面形态应是“控制台”，不是通用设置堆叠页。

### 3.2 允许能力（仅这些）
1. 悬浮窗始终前置预览。
2. 视频/音频设备手动选择（含 iPhone Continuity）。
3. 麦克风预览静音与实时电平。
4. 比例切换与自动放缩。

### 3.3 禁止能力（PiP 页面）
1. 录制入口。
2. 导出入口。
3. 语音转写入口。

### 3.4 默认行为
1. `enableCameraPiP = false`。
2. 冷启动不自动弹出 PiP 预览。
3. `bootstrap()` 仅刷新设备与同步状态，不自动启动预览。

### 3.5 顶部主按钮规则
1. 未显示预览时仅显示：`弹出 PiP 摄像`。
2. 已显示预览时仅显示：`收起 PiP 摄像`。
3. 点击 `弹出 PiP 摄像`：内部自动启用并执行预览弹出。
4. 点击 `收起 PiP 摄像`：仅隐藏预览，不自动反勾选内部启用状态。
5. 不再把内部启用状态作为用户主操作心智暴露。

### 3.6 比例规则
1. 比例固定：`自动 / 16:9 / 4:3`。
2. `自动`优先保证可见性与可拖拽性。
3. `16:9 / 4:3` 切换保持高度。

## 4. 状态隔离约束
1. 录屏状态文案仅写 `statusMessage`。
2. PiP 状态文案仅写 `pipStatusMessage`。
3. PiP 操作禁止覆写录屏主状态文案。
4. 菜单栏可同时展示 `录屏` 与 `PiP` 状态。

## 5. 窗口与设备约束
1. PiP 窗口需支持 `canJoinAllSpaces + fullScreenAuxiliary + moveToActiveSpace`。
2. Space/App 激活变化后需 `orderFrontRegardless` 兜底。
3. 设备枚举优先 DiscoverySession，空时回退 legacy API。
4. 设备断连不崩溃，需自动回退并给出可读提示。
5. 无权限/无设备禁止静默失败。

## 6. 录屏模块约束
1. 移除区域截取录屏能力，不再提供框选区域与固定虚线框。
2. 录屏仅支持 `macOS 主屏整屏录制`。
3. 点击开始录屏后立即隐藏主窗口。
4. 录制中通过独立悬浮控制器承载停止操作，不复用 PiP 窗口。
5. 悬浮控制器主体展示相机图标，可拖动。
6. 悬浮控制器标题栏需提供停止方块按钮，点击后结束录屏。
7. 正常停止、启动失败、异常停止都必须自动恢复主窗口。
8. `SCContentFilter` 需优先排除 `PJTool` 自身应用窗口；若排除失败，需给出可读提示。

## 7. 模块边界约束
1. `录屏` 模块继续承载录制控制与录制麦克风。
2. `视频处理` 模块继续承载拼接、剪切、导出。
3. PiP 迭代不得改变录屏状态机与视频处理语义。
4. 拼接语义保持“时间线插入拼接”，不得改为叠加语义。

## 8. 控制台布局约束（新增）
1. 左侧菜单栏默认启动宽度为 `200 px`。
2. 启动时必须强制应用默认宽度，不受历史分栏状态干扰。
3. 左侧菜单栏支持拖拽调整宽度，最小宽度不低于默认值。

## 9. 构建与校验（当前策略）
当前保持“门禁暂停”，但每次任务仍要求最小自检：
1. `xcodebuild -project PJTool.xcodeproj -scheme PJTool -destination 'platform=macOS' build`
2. `Scripts/run_logic_checks.sh`
3. 变更说明中列出：
   - 改动文件
   - 自检结果
   - 未覆盖风险
